# Vulnerability Disclosure Report: Subdomain Takeover

**Date:** May 02, 2026
**Author:** Zacheriah Alan Potter (Zeno Labs)
**Target Domain:** `pix.secretservice.gov`
**Vulnerability Type:** Subdomain Takeover via Dangling CNAME (CWE-400)
**Severity:** High

---

## 1. Description
During routine passive DNS reconnaissance, a dangling CNAME record was identified on the target domain. The domain `pix.secretservice.gov` currently points to `pix.trafficmanager.net`.

However, the Azure resource at `pix.trafficmanager.net` appears to be de-provisioned or deleted. This leaves the domain vulnerable to a Subdomain Takeover. A malicious actor could register the exact resource name within their own Azure tenant and effectively seize control of the content hosted at `pix.secretservice.gov`.

## 2. Steps to Reproduce
This finding was discovered passively. No active exploitation or resource claiming was performed, in strict adherence to safe harbor guidelines.
1. Perform a DNS query for the target domain:
   `dig pix.secretservice.gov CNAME +short`
2. Observe the output pointing to the vulnerable cloud provider:
   `pix.trafficmanager.net`
3. Verify via HTTP/HTTPS that the endpoint returns a standard "Resource Not Found" error indicative of a dangling cloud route.

## 3. Security Impact
If exploited, an attacker gains complete control over the subdomain. Because the domain carries the high-trust `.gov` top-level domain, the attacker could use this for:
* Launching highly credible spear-phishing campaigns.
* Bypassing Same-Origin Policy (SOP) to steal cookies or session tokens.
* Distributing malware masked as legitimate government software.

## 4. Remediation Recommendation
To resolve this vulnerability, please remove the dangling CNAME record (`pix.trafficmanager.net`) from the DNS zone file for `pix.secretservice.gov` immediately.
